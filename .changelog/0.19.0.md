### Added

- **New Components**
  - DataGrid-based List component with improved selection and grouping
  - TreeGrid support for hierarchical list structures
  - Skeleton loading states replacing legacy Shimmer components
  - v9 Dialog component replacing Modal
  - v9 Drawer component replacing Panel

- **Icon System**
  - ErrorCircle fallback icon for missing icons in catalog
  - Development mode console warnings for missing icons
  - Support for bundled filled/regular icon variants

- **Types & Enums**
  - `SelectionMode` enum (string-based: 'none', 'single', 'multiple')
  - `CheckboxVisibility` enum ('hidden', 'onHover', 'always')
  - Enhanced `IListColumn` interface with v9-compatible properties

- **Dependencies**
  - `@fluentui-contrib/react-tree-grid` ^0.3.2 for hierarchical lists

### Changed

- **Theme System**
  - Removed legacy `ThemeProvider` and `PortalCompatProvider`
  - Simplified to single `FluentProvider` (v9 only)
  - Body background and text colors now applied via useEffect
  - Removed `fonts.ts` and legacy theme complexity
  - Dark theme updated with GitHub-inspired colors

- **List Component** (Breaking Changes - See Migration Guide)
  - Migrated from `ShimmeredDetailsList` to `DataGrid` and `TreeGrid`
  - Removed `ListGroupHeader` and `ListHeader` subcomponents
  - Selection state management optimized (single-pass computation)
  - Removed props: `onItemContextMenu`, `onColumnResize`, `dragDropEvents`, `listProps`
  - `enableShimmer` now uses Skeleton instead of Shimmer
  - Column resizing not implemented (`isResizable` prop accepted but non-functional)

- **Icon Handling**
  - Replaced `getFluentIconWithFallback` with enhanced `getFluentIcon`
  - Icons now use v9 bundled icon system
  - Missing icons show ErrorCircle fallback instead of silent failure
  - All icon imports updated to `@fluentui/react-icons`

- **Component API Updates**
  - `Modal` ‚Üí `Dialog` with `open`/`onOpenChange` instead of `isOpen`/`onDismiss`
  - `Panel` ‚Üí `Drawer` with v9-compatible props
  - `Checkbox` label prop now supports complex ReactNode content
  - `ContextualMenu` ‚Üí `Menu` with v9 menu components
  - `format` function now uses `string-format` package (API-compatible)

- **Import Paths**
  - `DateRangeType`, `DayOfWeek` now from `@fluentui/react-calendar-compat`
  - `SelectionMode`, `CheckboxVisibility` now from `components/List/types`
  - All icon components from `@fluentui/react-icons`

### Fixed

- **Critical Issues**
  - Icon system now has proper fallback mechanism (no more null icons)
  - Theme component cleanup prevents memory leaks on unmount
  - Dialog `onDismiss` only called once on close (not multiple times)
  - List selection state optimized to prevent unnecessary re-renders

- **Performance**
  - Removed redundant `itemMeta` Map in List component
  - Combined three separate useMemo hooks into single-pass computation
  - Filter panel signature-based deduplication prevents infinite loops

- **UI Fixes**
  - ProjectList component rendering fixed for v9
  - Unmatched time entries rendering corrected
  - Mobile breadcrumb authentication check relocated
  - ScrollablePane replaced with CSS sticky positioning in LabelPicker
